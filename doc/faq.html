FAQ Sillaj

"DB: Syntax error" when I request a report
You are probably running an old version of MySQL server, which does not support the CAST... AS CHAR(8) syntax. You should upgrade your MySQL server or edit Report::getProject() and Report::getTask() in inc/sillaj.class.php (however you'll have date formatting issues)
I get a blank page
Generally it's because the PEAR components are not available. See in install.html :
  • Make sure you have a PEAR directory in your PHP directory and check that this PEAR directory is declared in the include_dir directive of php.ini. e.g. :
    include_path = ".;c:\php\includes;c:\php\pear;"
  • copy the content of sillaj/lib/pear/ in your PEAR directory if necessary
